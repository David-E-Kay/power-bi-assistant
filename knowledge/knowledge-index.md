# Knowledge Index

Routing manifest for project KB and skill files. Use this index FIRST when looking for a file — it tells you whether to read it directly or `Grep`-then-targeted-`Read`.

**Scope:** This index covers files under `knowledge/`, `artifacts/`, `.claude/skills/`, and `docs/` only. Plugin cache files (`~/.claude/plugins/cache/...`) are managed by their own skills and are out of scope.

---

## Access-Pattern Legend

| Pattern | Meaning |
|---|---|
| **direct-read** | Small enough to `Read` in full. |
| **grep-then-read** | Large generated artifact. `Grep` to locate the target span, then `Read` only that span with `offset`/`limit` — do NOT full-read. |
| **task-only** | Read only when the relevant task starts (skill files; not loaded by default). |
| **archived** | Deprecated; do not consult for active guidance. |

## External-Source Caches (`knowledge/confluence/`) — Optional

**Optional integration — empty by default.** Confluence caching is OFF until you configure it via the `confluence-cache` skill (`references/setup.md`). When enabled, cached pages are written to disk so they're searchable via `Grep`/`Read` alongside curated KB. Generated by skill workflows — do not hand-edit the `.md` files; use the owning skill. If you don't use Confluence, route any Confluence questions to the live Atlassian MCP instead.

| Folder | Source of truth | Owning skill | Access Pattern |
|---|---|---|---|
| `knowledge/confluence/` | `_manifest.yaml` (in same folder) | `.claude/skills/confluence-cache/SKILL.md` | **direct-read** (or `Grep` across the folder) |

Cached files have YAML frontmatter (`confluence_id`, `version`, `last_synced`) so freshness is visible without round-tripping the live MCP. For pages NOT in the cache, route to the live Atlassian MCP (`mcp__…__getConfluencePage`) directly — the skill is for cache management, not ad-hoc queries.

## Source-of-Truth Chain (Model Metadata)

When a task needs model metadata (relationships, measure definitions, dependencies), run this chain in order. State the chosen layer in the response so the user knows the staleness.

1. **Tabular Editor CLI** (live) — `where TabularEditor.exe` or `POWERBI_TE_CLI_PATH` env var. Preferred when available.
2. **Live TOM export** — `scripts/export_schema.py` (TE-free; model open in Power BI Desktop). Serializes the live model via TOM and runs the same parser, producing the same `artifacts/model-schema/model-schema-<slug>.md`. Preferred live fallback when TE CLI isn't installed. One-time prereq: `scripts/pbi_capture/provision_libs.py`.
3. **`scripts/bim_to_kb_markdown.py` snapshot** — `Glob artifacts/model-schema/*.md`. Offline `.bim` fallback. Flag staleness in responses.
4. **Power BI MCP server** — read `.mcp.json` and `~/.claude/.claude.json`; look for connectors named `power-bi`, `pbi`, `analysis-services`. Use for live queries it supports; TE CLI is preferred when both are available.

**Announce the source layer** in any analysis (e.g., "Source: parsed `.bim` snapshot, generated 2026-04-13 — live model may have diverged").

## Handling Large Files

For `artifacts/model-schema/<name>.md` and other large generated artifacts:

- `Grep` to locate the target object, then `Read` only that span with `offset`/`limit` — never full-read.
- Regenerate the snapshot via `bim_to_kb_markdown.py` when the schema changes (new tables, new measures, relationship topology changes).
- When a curated `knowledge/` file grows past the direct-read threshold (~500 lines or 50 KB), switch its access pattern to **grep-then-read** here.

Direct-read is correct for everything <500 lines.

## Updating This Index

Add a row when:

- Onboarding a new model (`artifacts/model-schema/model-schema-<name>.md`).
- Adding a new curated KB file under `knowledge/`.
- A previously direct-read file grows large enough to need grep-then-read.

Remove or move to "archived" when:

- A KB file is superseded (move the row's pattern to `archived` and add a "superseded by" note).
- A skill is split or removed.
