# Knowledge Index

Routing manifest for project KB and skill files. Use this index FIRST when looking for a file — it tells you whether to read directly or route through Context Mode.

**Scope:** This index covers files under `knowledge/`, `artifacts/`, `.claude/skills/`, and `docs/` only. Plugin cache files (`~/.claude/plugins/cache/...`) are managed by their own skills and are out of scope.

---

## Access-Pattern Legend

| Pattern | Meaning |
|---|---|
| **direct-read** | Small enough to `Read` in full. No indexing needed. |
| **Context Mode index/search** | Large generated artifact. Use `powerbi-context-mode` (`ctx_index` + `ctx_search`) — do NOT full-read. |
| **Context Mode one-shot** | Use `ctx_execute_file` for single-shot analysis (no need to keep the index). |
| **task-only** | Read only when the relevant task starts (skill files; not loaded by default). |
| **archived** | Deprecated; do not consult for active guidance. |

**Per-model naming convention:** Per-model behavioral KB files use a `{model}-` prefix — `{model}-gotchas.md`, `{model}-dax-performance.md`, `{model}-design-decisions.md` (e.g., `sales-gotchas.md`, `inventory-design-decisions.md`). They are **created on demand** when a model is onboarded — **none ship by default**. Files without a model prefix (`pbi-*`) are team-wide or transferable across models.

## External-Source Caches (`knowledge/confluence/`) — Optional

**Optional integration — empty by default.** Confluence caching is OFF until you configure it via the `confluence-cache` skill (`references/setup.md`). When enabled, cached pages are written to disk so they participate in `ctx_search` alongside curated KB. Generated by skill workflows — do not hand-edit the `.md` files; use the owning skill. If you don't use Confluence, route any Confluence questions to the live Atlassian MCP instead.

| Folder | Source of truth | Owning skill | Access Pattern | Re-index Trigger |
|---|---|---|---|---|
| `knowledge/confluence/` | `_manifest.yaml` (in same folder) | `.claude/skills/confluence-cache/SKILL.md` | **Context Mode index/search** (stable source name: `confluence-team-standards`) | After the `confluence-cache` skill reports any `updated` page |

Cached files have YAML frontmatter (`confluence_id`, `version`, `last_synced`) so freshness is visible without round-tripping the live MCP. For pages NOT in the cache, route to the live Atlassian MCP (`mcp__…__getConfluencePage`) directly — the skill is for cache management, not ad-hoc queries.

## Source-of-Truth Chain (Model Metadata)

When a task needs model metadata (relationships, measure definitions, dependencies), `powerbi-context-mode` runs this chain. See `.claude/skills/powerbi-context-mode/SKILL.md` for the full detection commands.

1. **Tabular Editor CLI** (live) — `where TabularEditor.exe` or `POWERBI_TE_CLI_PATH` env var. Preferred when available.
2. **`scripts/bim_to_kb_markdown.py` snapshot** — `Glob artifacts/model-schema/*.md`. Current operational fallback. Flag staleness in responses.
3. **Power BI MCP server** — read `.mcp.json` and `~/.claude/.claude.json`; look for connectors named `power-bi`, `pbi`, `analysis-services`. Use for live queries it supports; TE CLI is preferred when both are available.

The skill instructs Claude to **announce the source layer** in any analysis (e.g., "Source: parsed `.bim` snapshot, generated 2026-04-13 — live model may have diverged").

## When To Re-index

Trigger `ctx_index` against `artifacts/model-schema/<name>.md` (stable source name like `model-schema-<model>`):

- After `bim_to_kb_markdown.py` regenerates the schema (new tables, new measures, relationship topology changes).
- When a curated `knowledge/` file grows past the direct-read threshold (~500 lines or 50 KB) — at that point, add a Context Mode row here.

Do NOT re-index small curated files. Direct-read is correct for everything <500 lines.

---

## Updating This Index

Add a row when:

- Onboarding a new model (`artifacts/model-schema/model-schema-<name>.md`).
- Adding a new curated KB file under `knowledge/`.
- A previously direct-read file grows large enough to need Context Mode.

Remove or move to "archived" when:

- A KB file is superseded (move the row's pattern to `archived` and add a "superseded by" note).
- A skill is split or removed.
